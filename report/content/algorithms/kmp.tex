\subsection[The Knuth-Morris-Pratt algorithm]{The Knuth-Morris-Pratt algorithm (KMP algorithm)}
    \subsubsection{Introduction \& Application} %khai niem finite automaton trich tu introduction to algorithm
        \begin{itemize}%https://www.almabetter.com/bytes/articles/kmp-algorithm#%22Practical%20Applications%20of%20the%20KMP%20Algorithm%22 https://www.naukri.com/code360/library/kmp-string-matching-algorithm https://www.researchgate.net/publication/279442892_A_Survey_of_String_Matching_Algorithms https://www.reddit.com/r/learnprogramming/comments/1bk9ura/whats_the_best_stringmatching_algorithm_for/
            \item \textbf{Core idea:} % introduction-to-algorithms-3rd-edition
            The Knuth-Morris-Pratt (KMP) algorithm is a finite automaton which utilizes a prefix array (also known as the $\pi$). 
            For each substring $P[0 \dots \text{currentIndex}]$, the algorithm determines the longest proper prefix that is equal to a suffix of that same substring.            
            The construction of this array is an iterative process based on prior states. 
            When a mismatch occurs at the current index, the algorithm does not restart from the beginning; instead, it recursively consults previous prefix values until a matching character is found or the process reverts to the base state (0).
            \item \textbf{Real-world applications:}
            \begin{itemize}
                \item Text Editors: Using in "Search" or "Find and Replace" features to locate destination words.
                \item Bioinformatics: Searching for specific DNA.
                \item Checking for Plagiarism: Comparing documents to find and compare a script or a piece of code with the exact match in a large database.
            \end{itemize}
        \end{itemize}

        \subsubsection{Pseudocode}
            \textbf{Notation:}
            \begin{itemize}
                \item \textbf{Text $T$ (R rows, C columns)}: The 2D grid of characters.
                \item \textbf{Pattern $P$ (K keywords)}: The list of patterns to search for.
                \item \textbf{Prefix Array $\pi$}: An array storing the lengths of the longest proper prefix that is also a suffix.
            \end{itemize}

            \textsc{Convert-Grid-To-Strings}($T, R, C$)
            \begin{algorithmic}[1]
            \State $ans = \text{empty array}$
            \Statex
            \Comment{Extract Rows}
            \For{$i = 0$ \textbf{to} $R - 1$}
                \State $tmpString = \text{empty string}$
                \For{$j = 0$ \textbf{to} $C - 1$}
                    \State $tmpString = tmpString + T[i][j]$
                \EndFor
                \State \text{append } $tmpString$ \text{ to } $ans$
            \EndFor
            \Statex
            \Comment{Extract Columns}
            \For{$j = 0$ \textbf{to} $C - 1$}
                \State $tmpString = \text{empty string}$
                \For{$i = 0$ \textbf{to} $R - 1$}
                    \State $tmpString = tmpString + T[i][j]$
                \EndFor
                \State \text{append } $tmpString$ \text{ to } $ans$
            \EndFor
            \State \Return $ans$
            \end{algorithmic}

            \textsc{Calculate-Position}($idxString, m, R, idx$)
            \begin{algorithmic}[1]
            \If{$idxString < R$}
                \Statex
                \Comment{Horizontal match (Row extraction)}
                \State $start = (idxString, idx - 2 \cdot m)$
                \State $end = (idxString, idx - m - 1)$
            \Else
                \Statex
                \Comment{Vertical match (Column extraction)}
                \State $start = (idx - 2 \cdot m, idxString - R)$
                \State $end = (idx - m - 1, idxString - R)$
            \EndIf
            \State \Return $(start, end)$
            \end{algorithmic}

            \textsc{KMP-Algorithm}($T, P, K, R, C$)
            \begin{algorithmic}[1]
            \State $pos = \text{array of } K \text{ empty lists}$
            \State $List\_string = \textsc{Convert-Grid-To-Strings}(T, R, C)$
            \State $N = List\_string.length$
            \Statex
            \For{$idxString = 0$ \textbf{to} $N - 1$}
                \For{$pIdx = 0$ \textbf{to} $K - 1$}
                    \State $kmpString = P[pIdx] + \text{`\#'} + List\_string[idxString]$
                    \State $L = kmpString.length$
                    \State $m = P[pIdx].length$
                    \State $\pi = \text{array of size } L \text{ initialized to 0}$
                    \Statex
                    \Comment{Compute Prefix Function \& Search}
                    \For{$i = 1$ \textbf{to} $L - 1$}
                        \State $j = \pi[i - 1]$
                        \While{$j > 0$ \textbf{and} $kmpString[j] \neq kmpString[i]$}
                            \State $j = \pi[j - 1]$
                        \EndWhile
                        \If{$kmpString[j] == kmpString[i]$}
                            \State $j = j + 1$
                        \EndIf
                        \State $\pi[i] = j$
                        \Statex
                        \Comment{Match found}
                        \If{$\pi[i] == m$}
                            \State $matchPos = \textsc{Calculate-Position}(idxString, m, R, i)$
                            \State \text{append } $matchPos$ \text{ to } $pos[pIdx]$
                        \EndIf
                    \EndFor
                \EndFor
            \EndFor
            \State \Return $pos$
            \end{algorithmic}

    \indent \textbf{Notation:}
        \begin{itemize}
            \item \textbf{Text $T$ (length $n$)}: The main text being searched.
            \item \textbf{Pattern $P$}: The string that we are looking for.
            \item \textbf{Shift ($s$)}: The current offset of the text T.
            \item \textbf{Positions ([(start position, end position), \ldots])}: A list storing the indices where a complete match is found.
        \end{itemize}

    \indent \textbf{Example:} text $T = [b, a, n, a, b, a]$, pattern $P = [b, a, n, a]$ and string $S = [b, a, n, a, \#, b, a, n, a, b, a]$\\
    \indent Note: String $S$ is the concatenation of the pattern, the sentinel character $\#$, and the text. 
    \indent Any character can be used instead of $\#$, provided it does not appear in $T$ or $P$.

        \item Assuming we have calculated the $\pi$ array up to index 6, while the remaining indices are set to -1 (indicating they have not yet been calculated).\\
        Note that $\pi[0]$ is always $0$ because a single-character string has no proper prefixes.

            \centering
            \includegraphics[page = 2, width=0.8\textwidth, trim={2cm 2cm 0cm 2cm}, clip]{data/Illustrations/kmp.pdf}

        \end{figure}
        \item To find the answer for index 5, we must look at the previous result at $\pi[4]$ first. 
        Because $\pi[4]$ is 0, it tells us that there are no smaller matches to check, so we can stop looking back immediately. 
        Now, we only need to compare $S[5]$ with the very first letter $S[0]$. 
        Since both are 'b', they match, and we add 1 to get $\pi[5] = 1$.
        The calculation of the $\pi$ array depends only on the array itself, and character comparisons are performed only when necessary.\\ 
        Note: To make this process easier to understand, the author provides an illustration showing the comparison between the current prefix and suffix at each step.

        \begin{figure}[H]
            \centering
            \includegraphics[page = 3, width=0.8\textwidth, trim={2cm 2cm 0cm 2cm}, clip]{data/Illustrations/kmp.pdf}
            \caption{Step 3}
        \end{figure}
        \item Similar to step 2, we look at the previous result at $\pi[5]$ first.
        We see that $S[6]$ matches $S[1]$ (both are 'a'), so the search stops there.
        Because they match, we increase the value from 1 to 2; therefore, $\pi[6] = 2$.

        \begin{figure}[H]
            \centering
            \includegraphics[page = 6, width=0.8\textwidth, trim={2cm 2cm 0cm 2cm}, clip]{data/Illustrations/kmp.pdf}
            \caption{Step 4}
        \end{figure}

        \item We skip to index 9 because indices 7 and 8 follow the same logic as indices 5 and 6 in the previous step.
        Now, we continue with index 9. We do the same steps as before and get a mismatch, because $S[9]$ is 'b' but $S[4]$ is $'\#'$.
        According to the KMP algorithm, we find the next possible position until it matches with $S[9]$ or the value reaches 0.

        \begin{figure}[H]
            \centering
            \includegraphics[page = 7, width=0.8\textwidth, trim={2cm 2cm 0cm 2cm}, clip]{data/Illustrations/kmp.pdf}
            \caption{Step 5}
        \end{figure}
        \item Following the KMP algorithm's instruction, we try with $\pi[current\_value]$, which is $\pi[4]$. 
        This gives us a value of 0, so we stop there. 
        After finding the position, we must check whether that position matches the current index and we increase the value by 1 if it is a match. 
        Fortunately, $S[9]$ matches $S[0]$, so $\pi[9] = 1$.

        \begin{figure}[H]
            \centering
            \includegraphics[page = 8, width=0.8\textwidth, trim={2cm 2cm 0cm 2cm}, clip]{data/Illustrations/kmp.pdf}
            \caption{Step 6}
        \end{figure}
        \item In the same way as the previous step, we take the value from the last index 9, which is 1. 
        Because $S[10]$ matches $S[1]$, the algorithm stops looking back right now. 
        Moving to the comparison step, since we know $S[10]$ and $S[1]$ match, we increase the value from 1 to 2, resulting in $\pi[10] = 2$.\\
        We have now traversed to the end of array $S$, and the algorithm terminates. \\
        Based on the $\pi$ array, we find that there is 1 substring from the text $T$ that matches pattern $P$ by counting how many indices have a value equal to the length of $P$. 
        The matching substring starts at index 5 (index 8 minus the length of $P$ plus 1) and ends at index 8.
    \end{itemize}

    \subsubsection{Discussion}
    \begin{itemize}
        \item In scenario 1, the number of patterns $K$ is small, and the grid size is from $10 \times 10$ to $500 \times 500$. 
        The Knuth-Morris-Pratt algorithm is an efficient algorithm for this scenario because it ranks third among the five fastest algorithms. 
        \item In scenario 2, the number of patterns $K$ is large, and the text is fixed at $100 \times 100$. 
        In this case, the Knuth-Morris-Pratt algorithm takes more time to run because KMP is better for problems that don't have too many patterns to find. 
        So, it takes the second rank of the slowest algorithms in this test.
        \item Although scenario 1 has a larger theoretical time complexity than scenario 2, scenario 2 consumes more time in practice. 
        This is because the process of concatenating the grid strings and patterns must be done 20,000 times ($100 \cdot 100 \cdot 2$), while in case 1 it is only 5,000 times ($500 \cdot 2 \cdot 5$).
        \item In conclusion, the Knuth-Morris-Pratt algorithm is a suitable algorithm for string-matching, but it runs more efficiently when there are fewer patterns to find.
    \end{itemize}
